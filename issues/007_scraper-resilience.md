# Issue #007: eBay スクレイパーの耐障害性向上

## 概要
CSS セレクタに依存した HTML パースのため、eBay の DOM 変更で壊れるリスクがある。

## 現状
- BeautifulSoup + lxml で HTML パース
- 固定の CSS セレクタで要素取得
- リトライ（指数バックオフ）はあるがサーキットブレーカーなし

## やるべきこと
- [ ] セレクタの外部設定化（YAML or JSON）で変更時に再デプロイ不要に
- [ ] サーキットブレーカーパターンの導入（連続失敗時にスクレイピング停止）
- [ ] パース失敗時のアラート通知（LINE）
- [ ] スクレイピング結果の妥当性チェック（0件結果の検知等）
- [ ] eBay API (Finding API) への段階的移行検討

## 優先度
**低** — 現状動作しているが、長期運用で問題になる
